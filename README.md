Telegram Bot для онлайн-школы «КлассОнлайн»
 Описание проекта

Данный проект представляет собой Telegram-бота, предназначенного для автоматизации обработки заявок пользователей онлайн-школы.

Бот позволяет пользователям:

оставить заявку на занятие,

выбрать дату и время,

задать вопрос по курсам,

получить автоматические напоминания.

Для администратора реализована админ-панель, позволяющая:

просматривать новые и все заявки,

изменять статус заявок,

просматривать статистику.

Проект разработан в рамках производственной практики с целью получения практических навыков backend-разработки и работы с Telegram API.

 Используемые технологии

Python 3.11

aiogram 3.0

SQLite

Telegram Bot API

Git / GitHub

Railway (облачный хостинг)

 Основной функционал
Пользовательская часть:

Запуск бота через команду /start

Пошаговое заполнение заявки

Выбор даты и времени занятия

Отправка заявки администратору

Получение автоматических напоминаний

Административная часть:

Просмотр новых заявок

Просмотр всех заявок

Просмотр статистики

Отметка заявок как обработанных

Детальный просмотр информации по заявке

 Описание файлов
main.py

Точка входа в приложение. Запускает бота и инициализирует основные компоненты.

bot.py

Содержит основную логику работы бота:

обработку команд,

диалоги с пользователями,

работу с клавиатурами,

взаимодействие с базой данных.

admin_panel.py

Отдельный модуль для админ-панели:

команды администратора,

обработка inline-кнопок,

просмотр и управление заявками,

формирование статистики.

database.py

Реализует работу с базой данных SQLite:

создание таблиц,

добавление заявок,

получение и обновление данных,

подсчёт статистики.

utils.py

Вспомогательные функции:

генерация дат и времени,

форматирование данных,

упрощение основной логики бота.

 База данных

В проекте используется SQLite, так как она:

не требует отдельного сервера,

проста в настройке,

подходит для небольших проектов и прототипов.

База данных хранит:

заявки пользователей,

выбранные даты и время,

статус обработки заявок.

 Переменные окружения

Перед запуском проекта необходимо создать файл .env на основе .env.example и указать:

BOT_TOKEN=токен_telegram_бота
ADMIN_ID=id_администратора
